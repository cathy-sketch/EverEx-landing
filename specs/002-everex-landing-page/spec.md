# Feature Specification: Everex 랜딩 페이지

**Feature Branch**: `002-everex-landing-page`
**Created**: 2026-03-25
**Status**: Draft
**Input**: User description: "Everex 랜딩 페이지 구축. Everex는 AI 기반 근골격계 재활 디지털 헬스케어 기업(everex.co.kr). 핵심 제품은 MORA 플랫폼(MORA Vu, MORA Ex, MORA Care, MORA Cure, MORA Rehab). CES 2024 혁신상, iF/Red Dot 수상. 파트너: Samsung, LG, Novartis. 국문/영문 지원. 타겟: 의료기관, 기업 복지 담당자, 환자. CTA: 데모 신청. 기존 Next.js + TypeScript + Carbon Design System 스택 활용."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 방문자가 Everex의 가치를 즉시 이해한다 (Priority: P1)

의료기관 담당자, 기업 복지 담당자, 또는 환자가 처음 사이트를 방문했을 때, 히어로 섹션만 보고도 "Everex가 무엇인지, 왜 선택해야 하는지"를 30초 이내에 파악할 수 있다.

**Why this priority**: 랜딩 페이지의 핵심 목적은 방문자 이탈을 막고 가치를 빠르게 전달하는 것이다. 이 스토리가 실패하면 나머지 모든 섹션은 의미가 없다.

**Independent Test**: 히어로 섹션만 구현해도 핵심 메시지(태그라인, 서브카피, 주요 CTA)를 전달하는 독립적인 페이지로 배포 가능하다.

**Acceptance Scenarios**:

1. **Given** 방문자가 사이트에 처음 접속했을 때, **When** 스크롤 없이 히어로 섹션을 봤을 때, **Then** Everex의 핵심 가치(AI 근골격계 재활)와 데모 신청 버튼이 명확하게 보인다.
2. **Given** 방문자가 히어로 섹션을 보고 있을 때, **When** "데모 신청" CTA를 클릭하면, **Then** 데모 신청 폼 섹션으로 부드럽게 스크롤 이동한다.
3. **Given** 방문자가 국문/영문 언어를 전환하면, **Then** 히어로 섹션의 모든 텍스트가 해당 언어로 즉시 변경된다.

---

### User Story 2 - 방문자가 MORA 제품 라인업을 탐색한다 (Priority: P2)

방문자가 Everex의 5가지 MORA 제품(Vu, Ex, Care, Cure, Rehab)을 각각 어떤 문제를 해결하는지 이해하고, 자신에게 맞는 제품을 식별할 수 있다.

**Why this priority**: 제품 이해는 데모 신청으로 이어지는 핵심 전환 단계다. 타겟별로 다른 제품이 있으므로 맞춤 탐색이 전환율을 높인다.

**Independent Test**: 제품 섹션만 구현해도 5가지 제품 카드가 표시되고 각 제품의 대상/용도를 확인할 수 있는 독립 페이지로 동작 가능하다.

**Acceptance Scenarios**:

1. **Given** 방문자가 제품 섹션을 보고 있을 때, **When** 각 MORA 제품 카드를 볼 때, **Then** 제품명, 대상 사용자, 핵심 기능 요약이 표시된다.
2. **Given** 의료기관 담당자가 제품을 탐색할 때, **When** MORA Vu/Ex를 확인하면, **Then** 의료기관에 적합한 기능(AI 동작 분석, 처방 운동)이 강조되어 표시된다.
3. **Given** 기업 복지 담당자가 제품을 탐색할 때, **When** MORA Care를 확인하면, **Then** 임직원 복지 프로그램으로서의 가치가 명확히 전달된다.

---

### User Story 3 - 방문자가 신뢰도 지표를 확인하고 데모를 신청한다 (Priority: P3)

방문자가 수상 실적, 파트너사, 임상 검증 정보를 확인한 후 신뢰를 갖고 데모 신청 폼을 작성·제출한다.

**Why this priority**: 신뢰도 섹션은 구매 결정을 촉진하는 소셜 프루프다. P1, P2보다 후순위이지만 최종 전환(데모 신청)에 결정적 역할을 한다.

**Independent Test**: 신뢰도 섹션 + 데모 신청 폼만 구현해도 독립적인 전환 페이지로 동작한다.

**Acceptance Scenarios**:

1. **Given** 방문자가 신뢰도 섹션을 볼 때, **When** 수상 실적 카드(CES 2024, iF, Red Dot)와 파트너 로고(Samsung, LG, Novartis)를 확인하면, **Then** 각 항목의 이름과 간단한 설명이 표시된다.
2. **Given** 방문자가 데모 신청 폼을 작성할 때, **When** 이름, 소속, 이메일, 연락처, 메시지를 입력하고 제출하면, **Then** 성공 메시지가 표시된다.
3. **Given** 방문자가 폼을 작성하다 필수 항목을 빈칸으로 제출하면, **Then** 해당 항목에 오류 안내 메시지가 표시되고 제출이 차단된다.

---

### User Story 4 - 방문자가 언어를 전환한다 (Priority: P4)

한국어/영어 방문자 모두 자신의 언어로 전체 페이지를 탐색할 수 있다.

**Why this priority**: 글로벌 시장(미국, 싱가포르 등) 확장 중인 Everex에게 영문 지원은 필수이나, 핵심 기능 구현 후 추가 가능하므로 P4로 분류.

**Independent Test**: 언어 전환 버튼을 구현하고 전체 텍스트가 두 언어로 정상 표시되는지 확인.

**Acceptance Scenarios**:

1. **Given** 방문자가 언어 전환 버튼(KO/EN)을 클릭하면, **When** 페이지 전체의 텍스트가 선택한 언어로 변경되고, **Then** 레이아웃이 깨지지 않는다.
2. **Given** 방문자가 영문으로 전환한 상태에서 페이지를 새로고침하면, **Then** 선택한 언어 설정이 유지된다.

---

### Edge Cases

- 데모 신청 폼 제출 시 네트워크 오류가 발생하면 사용자에게 재시도 안내를 표시한다.
- 모바일/태블릿 화면에서 제품 카드가 올바르게 반응형으로 표시된다.
- 파트너 로고 이미지 로드 실패 시 대체 텍스트가 표시된다.
- 언어 전환 시 이미지 alt 텍스트도 해당 언어로 제공된다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 히어로 섹션에 Everex의 핵심 가치 태그라인, 서브카피, "데모 신청" CTA 버튼을 표시해야 한다.
- **FR-002**: 시스템은 MORA 플랫폼 5개 제품(Vu, Ex, Care, Cure, Rehab)을 각각의 카드 형태로 표시해야 한다.
- **FR-003**: 각 제품 카드는 제품명, 대상 사용자(의료기관/기업/환자), 핵심 기능 요약을 포함해야 한다.
- **FR-004**: 시스템은 수상 실적(CES 2024 혁신상, iF Design Award, Red Dot Award) 섹션을 표시해야 한다.
- **FR-005**: 시스템은 파트너 로고(Samsung, LG, Novartis 등) 섹션을 표시해야 한다.
- **FR-006**: 시스템은 이름, 소속, 이메일(필수), 연락처, 메시지 필드를 포함한 데모 신청 폼을 제공해야 한다.
- **FR-007**: 데모 신청 폼은 필수 항목 미입력 및 이메일 형식 오류 시 인라인 오류 메시지를 표시해야 한다.
- **FR-008**: 데모 신청 폼 제출 성공 시 확인 메시지를 표시해야 한다.
- **FR-009**: 사용자는 페이지 상단 내비게이션에서 한국어/영어를 전환할 수 있어야 한다.
- **FR-010**: 언어 전환 시 페이지의 모든 텍스트 콘텐츠가 선택한 언어로 변경되어야 한다.
- **FR-011**: 내비게이션 메뉴는 제품, 수상/파트너, 데모 신청 섹션으로의 앵커 링크를 포함해야 한다.
- **FR-012**: 페이지는 모바일, 태블릿, 데스크톱 화면 크기에서 정상적으로 표시되어야 한다.

### Key Entities

- **제품(Product)**: MORA 플랫폼의 개별 제품. 속성: 이름, 대상 사용자, 핵심 기능 설명, 아이콘/이미지.
- **수상 실적(Award)**: Everex가 획득한 수상 정보. 속성: 수상명, 수여 기관, 연도, 카테고리.
- **파트너(Partner)**: 전략적/글로벌/헬스케어 파트너사. 속성: 파트너명, 로고, 파트너 유형.
- **데모 신청(DemoRequest)**: 방문자가 제출하는 데모 요청 정보. 속성: 이름, 소속, 이메일, 연락처, 메시지.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 방문자가 히어로 섹션에서 "데모 신청" CTA를 클릭해 폼 섹션에 도달하는 데 30초 이내에 완료 가능하다.
- **SC-002**: 데모 신청 폼을 처음 사용하는 방문자가 3분 이내에 폼을 완성하고 제출할 수 있다.
- **SC-003**: 페이지의 모든 핵심 섹션(히어로, 제품, 수상, 파트너, 폼)이 모바일/태블릿/데스크톱 3가지 화면 크기에서 레이아웃 깨짐 없이 표시된다.
- **SC-004**: 언어 전환 버튼 클릭 후 1초 이내에 전체 페이지 텍스트가 해당 언어로 변경된다.
- **SC-005**: 폼 유효성 검사 오류 메시지가 해당 필드 바로 아래에 명확하게 표시된다.

## Assumptions

- 데모 신청 폼 제출 데이터는 백엔드 이메일 발송 또는 외부 CRM 연동 없이, 제출 성공 메시지만 표시하는 MVP 범위로 구현한다 (실제 연동은 후속 작업).
- 파트너 로고 및 수상 배지 이미지 에셋은 텍스트 기반으로 대체하거나 공개 자료를 참조한다.
- 언어 전환은 페이지 내 상태로 관리하며, 별도의 URL 경로(/en, /ko) 분리는 v1 범위 밖이다.
- 모바일 화면 지원은 필수이나, 네이티브 앱 수준의 최적화는 v1 범위 밖이다.
- 사용자 인증(로그인) 기능은 이 랜딩 페이지 범위에 포함되지 않는다.
- 기존 Next.js App Router + TypeScript 프로젝트 구조를 활용하며, 기존 게시판 관련 코드는 별도로 유지된다.
